## 2025-05-18 - [Command Injection via execSync]
**Vulnerability:** Found multiple instances where user-controlled inputs (`projectDir`, `filePath`) were interpolated directly into shell commands via `execSync` (e.g., `execSync(\`node "\${cliPath}" init --dir "\${projectDir}"\`)`).
**Learning:** This is a classic command injection vulnerability. If `projectDir` contains shell metacharacters like `;`, `&&`, or `||`, it allows arbitrary command execution.
**Prevention:** Always use `execFileSync` (or `execFile` for async) instead of `execSync` when executing commands with dynamic inputs. Pass arguments as an array to ensure they are passed directly to the executable without shell interpolation.

## 2025-05-18 - [Command Injection via execSync in VS Code Extension]
**Vulnerability:** Found a critical command injection vulnerability in `vscode-extension/extension.js` where the user-controlled `workspaceDir` path was interpolated directly into a shell command via `execSync` (`cmd = \`"\${localBin}" \${args}\`; execSync(cmd)`). An attacker could open a workspace with a maliciously crafted directory name (e.g., `foo" && echo VULNERABLE && echo "bar`) to achieve arbitrary code execution.
**Learning:** This is a classic path-based command injection vulnerability. VS Code extensions must be extremely careful when executing external commands based on workspace paths, as these paths are implicitly controlled by the user.
**Prevention:** Always use `execFileSync` instead of `execSync` when executing commands with dynamic inputs derived from file paths. Pass arguments as an array to ensure they are passed directly to the executable without shell interpolation. If `execSync` must be retained on Windows (e.g., for `.cmd` files), strictly validate the path against dangerous shell metacharacters (`/[&|;"'\`\n]/`) before execution.

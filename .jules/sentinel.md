## 2025-05-18 - [Command Injection via execSync]
**Vulnerability:** Found multiple instances where user-controlled inputs (`projectDir`, `filePath`) were interpolated directly into shell commands via `execSync` (e.g., `execSync(\`node "\${cliPath}" init --dir "\${projectDir}"\`)`).
**Learning:** This is a classic command injection vulnerability. If `projectDir` contains shell metacharacters like `;`, `&&`, or `||`, it allows arbitrary command execution.
**Prevention:** Always use `execFileSync` (or `execFile` for async) instead of `execSync` when executing commands with dynamic inputs. Pass arguments as an array to ensure they are passed directly to the executable without shell interpolation.
## 2023-10-25 - [Command Injection in VS Code Extension]
**Vulnerability:** The VS Code extension used `execSync(cmd, { cwd: workspaceDir })` to execute `specguard` commands, passing user-controlled directory paths (via `workspaceDir`) and string-interpolated arguments directly. This allows an attacker to execute arbitrary commands if a malicious workspace directory is opened.
**Learning:** VS Code extension commands executing in the workspace context must treat the workspace path as untrusted user input, as it can contain shell metacharacters or spaces, leading to command injection and RCE. Also, `execFileSync` cannot natively run `.cmd` files on Windows without `shell: true`, which re-introduces RCE.
**Prevention:** Always use `execFileSync` with array arguments instead of `execSync` with concatenated strings. For Node scripts on Windows, execute the `node` binary directly with the script path as an argument to bypass the need for `shell: true`.

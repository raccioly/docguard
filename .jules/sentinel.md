## 2024-05-24 - [Fix Command Injection in execSync]
**Vulnerability:** Command injection vulnerability identified in `execSync` calls in `cli/validators/doc-quality.mjs` and `cli/validators/freshness.mjs`. Unsanitized user inputs or file paths interpolated directly into the command string could allow an attacker to execute arbitrary shell commands.
**Learning:** This codebase relies entirely on Node.js built-ins and uses `node:child_process` `execSync` extensively for sub-shell execution without dependencies. Utilizing `execSync` with strings allows shell interpolation which exposes the code to injection attacks via paths with special shell characters.
**Prevention:** Replace `execSync` with `execFileSync` when incorporating dynamic inputs such as file paths. `execFileSync` executes the program directly, accepting arguments as an array, completely bypassing the shell so special characters like `;`, `&`, or `$()` are correctly treated as literal strings. When dealing with `git` commands, utilize `--` to prevent argument option injection.

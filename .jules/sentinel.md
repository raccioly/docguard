## 2024-05-23 - Command Injection in Spec Kit Agent Detection
**Vulnerability:** A critical Command Injection vulnerability existed where `specify init` was called via `execSync()`. The script interpolated the value of `--ai` (derived from `detectAIAgent`, which reads `init-options.json` from the `.specify/` directory) into a shell command without escaping or validating the input. This could be exploited by dropping a malicious `.specify/init-options.json` file in a repository.
**Learning:** `execSync()` executes a command inside a shell environment by default. When dynamically parsed configuration values are concatenated into the command string, it inherently exposes the CLI to command injection attacks, even if the value seems like a simple config string (e.g., an agent ID).
**Prevention:** Always use `execFileSync` (or `spawnSync` without the `shell: true` option) and provide arguments as an array to avoid shell evaluation. As defense-in-depth, strictly validate dynamically loaded inputs using allowlist regexes (e.g., `/^[a-zA-Z0-9-]+$/`) before using them in cross-platform process execution (since tools like `cmd.exe /c` still process metacharacters).

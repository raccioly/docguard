## 2024-04-10 - Standardize CLI Error Outputs\n**Learning:** In a pure CLI tool like this, traditional web UX doesn't apply. The user experience here is the Developer Experience (DX). Printing fatal startup configuration errors to standard output (`console.log`) pollutes log files during shell redirection. Outputting them to standard error (`console.error`) is the correct UX/DX pattern.\n**Action:** Always route CLI-blocking or configuration errors to `console.error` (stderr) instead of `console.log`.
